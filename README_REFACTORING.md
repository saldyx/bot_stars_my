# Рефакторинг Pixel Stars Bot

## Новая структура проекта

Проект был профессионально разделен на модульную архитектуру для улучшения читаемости, поддержки и масштабируемости кода.

### 📁 Структура директорий

```
/workspace/
├── config/                 # Конфигурация
│   ├── __init__.py
│   ├── database.py         # Настройки подключения к БД
│   └── states.py           # FSM состояния
│
├── models/                 # Модели данных
│   ├── __init__.py
│   ├── user.py            # Модель пользователей
│   ├── referral.py        # Модель реферальной системы
│   ├── task.py            # Модели заданий
│   ├── channel.py         # Модель каналов
│   ├── crypto.py          # Модели криптоплатежей
│   ├── game.py            # Модели игр
│   ├── utm.py             # Модели UTM и статистики
│   ├── withdrawal.py      # Модели выводов и промокодов
│   └── database_init.py   # Инициализация всех таблиц
│
├── services/              # Бизнес-логика
│   ├── __init__.py
│   ├── user_service.py    # Сервис пользователей
│   ├── task_service.py    # Сервис заданий
│   ├── payment_service.py # Сервис платежей
│   ├── game_service.py    # Сервис игр
│   ├── channel_service.py # Сервис каналов
│   ├── utm_service.py     # Сервис UTM
│   ├── external_api.py    # Внешние API (SubGram, GramAds)
│   └── subscription_service.py # Сервис подписок
│
├── handlers/              # Обработчики команд
│   ├── __init__.py
│   ├── start_handler.py   # Обработчик /start
│   ├── menu_handler.py    # Обработчики меню
│   ├── admin_handler.py   # Админские команды
│   └── subscription_handler.py # Обработчики подписок
│
├── utils/                 # Утилиты
│   ├── __init__.py
│   ├── captcha.py         # Генерация капчи
│   ├── helpers.py         # Вспомогательные функции
│   ├── middleware.py      # Middleware для бота
│   └── flyer_integration.py # Интеграция с Flyer API
│
├── main.py               # Основной файл (рефакторинг)
├── database.py           # Старая база данных (сохранена)
├── database_refactored.py # Новая база с обратной совместимостью
└── settings.py           # Настройки бота
```

## 🔄 Обратная совместимость

Новая архитектура полностью совместима со старым кодом:

- **`database_refactored.py`** - содержит все старые функции как алиасы к новым методам
- **`main.py`** - обновлен с новыми импортами, но функциональность сохранена
- **Старые файлы** - сохранены как `main_old.py` и `database_old.py`

## 🚀 Преимущества новой структуры

### 1. **Модульность**
- Каждая функциональность вынесена в отдельный модуль
- Легко найти и изменить нужную логику
- Возможность независимого тестирования модулей

### 2. **Разделение ответственности**
- **Models** - работа с данными
- **Services** - бизнес-логика
- **Handlers** - обработка команд
- **Utils** - вспомогательные функции

### 3. **Масштабируемость**
- Легко добавлять новые функции
- Простое расширение существующих модулей
- Четкая архитектура для больших команд

### 4. **Поддержка**
- Код легче читать и понимать
- Быстрое исправление ошибок
- Простое добавление новых возможностей

## 📝 Миграция

### Для запуска с новой архитектурой:
```bash
python main.py
```

### Для возврата к старой версии:
```bash
mv main.py main_new.py
mv main_old.py main.py
mv database.py database_new.py  
mv database_old.py database.py
```

## 🛠 Основные изменения

1. **База данных разделена на модели** по функциональности
2. **Бизнес-логика вынесена в сервисы**
3. **Обработчики команд разделены по типам**
4. **Утилиты и middleware выделены отдельно**
5. **Конфигурация централизована**
